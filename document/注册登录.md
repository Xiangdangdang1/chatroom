### 知识点

- ```cpp
  if (_login_dlg) {
      delete _login_dlg;
      _login_dlg = nullptr;
  }
  ```

  如果只是释放对象，没有置空指针，`_login_dlg`仍然保存之前的地址，但是地址保存的内容是空，后续如果使用这个指针可能会很危险

- 头文件的`signals`，`public slots`的写法

  Qt中的信号与槽，槽是普通的 C++ 成员函数，可以被正常调用，它们唯一的特殊性就是很多信号可以与其相关联。槽的参数不能有缺省值。

  通过调用`connect`函数来将某个对象的信号与另外一个对象的槽函数相关联

  `QObject::connect(sender, &SenderClass::signal, receiver, &ReceiverClass::slot);`

  参数列表

  | **参数**                  | **类型**             | **说明**                                         |
  | ------------------------- | -------------------- | ------------------------------------------------ |
  | `sender`                  | `QObject*`           | 发送信号的对象                                   |
  | `signal`                  | **函数指针**         | 发送对象的信号，格式：`&SenderClass::signalName` |
  | `receiver`                | `QObject*`           | 接收信号的对象                                   |
  | `slot`                    | **函数指针**         | 槽函数，格式：`&ReceiverClass::slotName`         |
  | `connectionType` _(可选)_ | `Qt::ConnectionType` | 连接类型（默认 `Qt::AutoConnection`）            |

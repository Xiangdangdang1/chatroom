## 后端

在`mainwindow.cpp`中如果`new`了新创建的对象(`_login_dlg`等)，没有指定父节点，QT不会帮我们删除，要手动处理或用智能指针。**QT的对象树机制**，可以设置父窗口`_login_dlg = new LoginDialog(this);`，当父对象`this`销毁，递归`delete`子对象。

以下部分就不用写了

```cpp
if (_login_dlg) {
    delete _login_dlg;
    _login_dlg = nullptr;
}

if (_reg_dlg) {
    delete _reg_dlg;
    _reg_dlg = nullptr;
}
```





**通过`mainwindow`连接`LoginDialog`与`RegisterDialog`的流程**

1. 登录界面有一个`reg_btn`按钮，`connect(ui->reg_btn, &QPushButton::clicked, this, &LoginDialog::switchRegister)`

   连接按钮事件，点击`reg_btn`按钮，发出`click`信号，`LoginDialog`接收该信号，同时发送自己定义的信号`switchRegister`给`mainwindow`

2. `mainwindow`接收信号，`slot`执行操作，切换到注册界面

   `connect(_login_dlg, &LoginDialog::switchRegister, this, &MainWindow::SlotSwitchReg);`





### 注册

1. 通过单例模式发送http请求，这是异步请求

   ```cpp
   HttpMgr::GetInstance()->PostHttpReq()
   ```

2. 发送请求结束之后，调用回调函数，处理成功/失败，无论是否成功都会发送信号`sig_http_finish`，给http模块

   ```cpp
   QObject::connect(reply, &QNetworkReply::finished, [self, reply, req_id, mod](){
           //处理错误情况
           if (reply->error() != QNetworkReply::NoError)
           {
               qDebug() << reply->errorString();
               //发送信号通知完成
               emit self->sig_http_finish(req_id, "", ErrorCodes::ERR_NETWORK, mod);
               reply->deleteLater();
               return;
           }
   
           //无错误
           QString res = reply->readAll();
           //发送信号通知完成
           emit self->sig_http_finish(req_id, res, ErrorCodes::SUCCESS, mod);
           reply->deleteLater();
           return;
       });
   }
   ```

3. 找到信号对应的槽函数，这个槽函数发送信号说明http响应结束

   ```cpp
   connect(this, &HttpMgr::sig_http_finish, this, &HttpMgr::slot_http_finish);
   
   //槽函数
   void HttpMgr::slot_http_finish(ReqId req_id, QString res, ErrorCodes err, Modules mod)
   {
       //如果是注册模块
       if (mod == Modules::REGISTERMOD)
       {
           //发送信号通知指定模块的http的响应结束了
           emit sig_reg_mod_finish(req_id, res, err);
       }
   }
   ```

4. 再找到对应的槽函数，发现是通过`_handler`处理

   ```cpp
   connect(HttpMgr::GetInstance().get(), &HttpMgr::sig_reg_mod_finish, this, &RegsisterDialog::slot_reg_mod_finish);
   ```

5. 在`void RegsisterDialog::initHttpHandlers()`新注册用户逻辑





## 知识点

- ```cpp
  if (_login_dlg) {
      delete _login_dlg;
      _login_dlg = nullptr;
  }
  ```

  如果只是释放对象，没有置空指针，`_login_dlg`仍然保存之前的地址，但是地址保存的内容是空，后续如果使用这个指针可能会很危险

- 头文件的`signals`，`public slots`的写法

  Qt中的信号与槽，槽是普通的 C++ 成员函数，可以被正常调用，它们唯一的特殊性就是很多信号可以与其相关联。槽的参数不能有缺省值。

  通过调用`connect`函数来将某个对象的信号与另外一个对象的槽函数相关联

  `QObject::connect(sender, &SenderClass::signal, receiver, &ReceiverClass::slot);`

  参数列表

  | **参数**                  | **类型**             | **说明**                                         |
  | ------------------------- | -------------------- | ------------------------------------------------ |
  | `sender`                  | `QObject*`           | 发送信号的对象                                   |
  | `signal`                  | **函数指针**         | 发送对象的信号，格式：`&SenderClass::signalName` |
  | `receiver`                | `QObject*`           | 接收信号的对象                                   |
  | `slot`                    | **函数指针**         | 槽函数，格式：`&ReceiverClass::slotName`         |
  | `connectionType` _(可选)_ | `Qt::ConnectionType` | 连接类型（默认 `Qt::AutoConnection`）            |

- Qt的正则表达式`QRegularExpression regex(R"((\w+)(\.|_)?(\w*)@(\w+)(\.(\w+))+)");`

  - 头文件`#include <QRegularExpression>`

  - `R"( ... )"`，表示C++的原始字符串字面值，避免`\`等转义字符
  - `(\w+)`，`+`**至少有一个**字符匹配a~z，A~Z，0~9
  -  `(\.|_)?`，`.`或`_`可能出现，`?`0次或1次
  - `*`，0个或多个

- 回顾`extern`的用法
